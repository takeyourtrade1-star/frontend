DISABILITA TEMPORANEAMENTE INVIO EMAIL NELLA REGISTRAZIONE

PROBLEMA: La registrazione fallisce con errore 500 probabilmente a causa della configurazione SMTP mancante.

SOLUZIONE TEMPORANEA: Disabilita l'invio email durante la registrazione per permettere la registrazione degli utenti. L'email verrà integrata successivamente.

AZIONI RICHIESTE:

1. Trova il controller/metodo che gestisce POST /api/auth/register

2. Cerca dove viene inviata l'email di verifica durante la registrazione (probabilmente dopo la creazione dell'utente)

3. Modifica il codice per:
   - Avvolgere l'invio email in un try-catch che NON blocca la registrazione
   - Loggare l'errore email ma continuare comunque
   - Oppure commentare temporaneamente l'invio email

ESEMPIO DI MODIFICA:

PRIMA (probabilmente):
```php
// Crea utente
$user = User::create([...]);

// Crea profilo
$user->profile()->create([...]);

// Invia email (questo probabilmente fallisce e blocca tutto)
Mail::to($user->email)->send(new VerificationEmail($user));
```

DOPO (modifica richiesta):
```php
// Crea utente
$user = User::create([...]);

// Crea profilo
$user->profile()->create([...]);

// Invia email (NON bloccare se fallisce)
try {
    Mail::to($user->email)->send(new VerificationEmail($user));
    \Log::info('Verification email sent', ['user_id' => $user->id]);
} catch (\Exception $e) {
    // Logga l'errore ma NON bloccare la registrazione
    \Log::warning('Failed to send verification email', [
        'user_id' => $user->id,
        'email' => $user->email,
        'error' => $e->getMessage()
    ]);
    // Continua comunque - la registrazione è andata a buon fine
}
```

OPPURE (soluzione ancora più semplice - commenta temporaneamente):
```php
// Crea utente
$user = User::create([...]);

// Crea profilo
$user->profile()->create([...]);

// TODO: Riabilitare invio email quando SMTP sarà configurato
// Mail::to($user->email)->send(new VerificationEmail($user));
\Log::info('Registration successful - Email sending temporarily disabled', ['user_id' => $user->id]);
```

4. Verifica che la risposta di successo venga comunque inviata anche se l'email non viene inviata

5. Assicurati che l'utente venga comunque creato e autenticato correttamente

RISULTATO ATTESO:
- La registrazione funziona anche senza configurazione SMTP
- L'utente viene creato correttamente
- Il token viene restituito
- L'errore email viene solo loggato, non blocca la registrazione

NOTA: Secondo la documentazione, l'invio email dovrebbe già essere asincrono e non bloccante, ma evidentemente nel codice c'è qualcosa che lo blocca. Questa modifica permetterà di procedere e poi integreremo l'email correttamente.
