Analizza il problema di registrazione utenti nel backend Laravel/Lumen.

PROBLEMA: L'endpoint POST /api/auth/register restituisce sempre errore 500 con risposta:
{
  "success": false,
  "message": "Errore durante la registrazione",
  "error": null
}

DATI INVIATI (corretti):
{
  "account_type": "personal",
  "country": "IT",
  "phone_prefix": "+39",
  "phone": "123456789",
  "email": "test@example.com",
  "username": "testuser",
  "password": "password123",
  "password_confirmation": "password123",
  "first_name": "Mario",
  "last_name": "Rossi"
}

AZIONI RICHIESTE:
1. Trova il controller/metodo che gestisce /api/auth/register
2. Controlla i log Laravel (storage/logs/laravel.log) per l'ultimo errore 500
3. Verifica se c'è un try-catch che nasconde l'errore reale (error: null)
4. Controlla:
   - Validazione dei campi (soprattutto account_type='private', first_name/last_name)
   - Creazione utente e profilo (tabella user_profiles)
   - Configurazione email SMTP (nei log c'era "SMTP configuration incomplete")
   - Transazioni database che potrebbero fallire
   - Eventi/Listener che vengono triggerati
5. Abilita temporaneamente APP_DEBUG=true per vedere l'errore completo
6. Aggiungi logging dettagliato nel catch block per vedere l'errore reale

FORNISCI:
- Stack trace completo dell'errore
- File e riga dove avviene l'errore
- Messaggio errore specifico
- Causa root del problema
- Fix proposto con codice
